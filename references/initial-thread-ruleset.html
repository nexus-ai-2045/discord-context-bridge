任意割り込み

追加文脈

過去ログ、rule、link、attachment、image、transcript の取得へ戻り、理解と risk check を再実行する。

直接編集

人間が Markdown や返信候補を直接編集できる。編集済み本文を次の review の source of truth にする。

Stopline

  • Discord への送信、reaction、edit、delete、account action 自動化はしない。
  • 全 server / channel を巡回しない。
  • token、cookie、browser profile、webhook URL を読まない、保存しない。
  • raw private Discord text や local evidence artifact を public package に入れない。
  • Note 記事作業や note-publishing-suite verifier 修復をこの scope に混ぜない。

境界マップ

Public core
safe schema、件数、label、risk state、review metadata、context budget、handoff packet 構造。
Private adapter
Chrome 観測、download、文字起こし、OneFile capture、local evidence handling。public package の外で扱う。
Human role
thread を選ぶ。Markdown review を読む。必要なら直接編集する。copy、待機、修正、停止を判断する。
この可視化は ruleset の review artifact です。Markdown SSOT は references/initial-thread-ruleset.md のままです。
